API Reference

parseMeasurement(input, options?): ParsedValue | null

Parse a string input into a structured measurement object. Handles both metric and imperial units for height and weight with smart unit detection.

Parameters

  • input: string - The measurement string to parse (e.g., "6 ft", "150 lbs", "180 cm")
  • options?: ParseOptions - Optional configuration object

ParseOptions

  • type?: 'height' | 'weight' - Specify the measurement type to parse
  • fuzziness?: number - Tolerance level for fuzzy unit matching
  • allowUnqualified?: boolean - Allow parsing numbers without explicit units
  • inferUnit?: 'metric' | 'imperial' - System to use when inferring units for unqualified values
  • normalizedUnit?: Unit - Convert all matches to this unit in the output

ParsedValue

  • value: number - The numeric value of the measurement (normalized if normalizedUnit is specified)
  • unit: Unit - The unit of measurement (normalized if normalizedUnit is specified)
  • type: MeasurementType - The type of measurement ('height' or 'weight')
  • matches: Match[] - Array of individual measurement components
  • raw: string - The complete input string

Match

  • value: number - The numeric value of this component
  • unit: Unit | null - The unit of this component

Example Usage

Here are common usage patterns with expected results:

TypeScript 
// example.ts
import { parseMeasurement } from '@afahy/fluent-measures';

// --- Basic Usage ---

// Height measurements
// Simple height measurement
const heightFt = parseMeasurement('6 ft');
// ‚úì Returns: { value: 6, unit: 'ft', type: 'height', matches: [{ value: 6, unit: 'ft' }], raw: '6 ft' }

// Complex height with components and normalization to centimeters
const heightCm = parseMeasurement('5\' 11"', {
  type: 'height',
  normalizedUnit: 'cm'
});
/* ‚úì Returns:
{
  value: 180.34,
  unit: 'cm',
  type: 'height',
  matches: [
    { value: 5, unit: 'ft', match: '5\'' },
    { value: 11, unit: 'in', match: '11"' }
  ],
  raw: '5\' 11"'
}
*/

// Same height with components but normalized to inches
const heightIn = parseMeasurement('5\' 11"', {
  type: 'height',
  normalizedUnit: 'in'
});
/* ‚úì Returns:
{
  value: 71,
  unit: 'in',
  type: 'height',
  matches: [
    { value: 5, unit: 'ft', match: '5\'' },
    { value: 11, unit: 'in', match: '11"' }
  ],
  raw: '5\' 11"'
}
*/

// Basic weight measurement in pounds
const weightLbs = parseMeasurement('150 lbs');
// ‚úì Returns: { value: 150, unit: 'lb', type: 'weight', matches: [{ value: 150, unit: 'lb' }], raw: '150 lbs' }

// Weight in kilograms with spelled-out text
const weightKg = parseMeasurement('eighty kilograms', { type: 'weight' });
// ‚úì Returns: { value: 80, unit: 'kg', type: 'weight', matches: [{ value: 80, unit: 'kg' }], raw: 'eighty kilograms' }

// Convert weight to kilograms
const weightConverted = parseMeasurement('150 lbs', {
  type: 'weight',
  normalizedUnit: 'kg'
});
// ‚úì Returns: { value: 68.04, unit: 'kg', type: 'weight', matches: [{ value: 150, unit: 'lb' }], raw: '150 lbs' }

// Unit inference for unqualified values
const inferredHeight = parseMeasurement('72', {
  type: 'height',
  allowUnqualified: true,
  inferUnit: 'imperial'
});
// ‚úì Returns: { value: 72, unit: 'in', type: 'height', matches: [{ value: 72, unit: 'in' }], raw: '72' }

Troubleshooting

Null Return Values

If parseMeasurement() returns null, check:

  • Input string format matches supported patterns
  • Units are supported (see Supported Units section)
  • Numbers are valid and not out of reasonable range
Unexpected Unit Types

When getting unexpected unit types:

  • Specify the type option explicitly
  • Check for ambiguous unit abbreviations
  • Ensure proper spacing between number and unit

Tips & Best Practices

  • Always handle null returns in your code
  • Use the fuzziness option carefully - higher values may lead to incorrect matches
  • When working with unqualified values, always set both allowUnqualified and inferUnit options
  • Consider input sanitization for user-provided values
